# Learnings

This file accumulates discoveries across iterations. Every time an agent
completes a task, it should append to the relevant section below with
anything future iterations should know:

- Gotchas discovered
- Non-obvious conventions in this codebase
- Commands that work vs. commands that seem right but don't
- Patterns to follow vs. anti-patterns to avoid

**Every session reads this before starting work.** The whole point of this
file is that hard-won knowledge from iteration N is available (without
re-discovery) at iteration N+1.

Rules:
- **Append only.** Never rewrite or delete existing entries.
- Keep entries short. A single bullet point is often enough.
- Date-stamp every entry so stale knowledge is easy to spot: `[YYYY-MM-DD]`.
